## 命名規約・タグ規約とは

プロジェクト全体で一貫性のある命名規則とタグ付けルールを定めることで、コードの可読性・保守性・検索性を向上させます。すべてのファイル、コンポーネント、変数、タグは以下の規約に従う必要があります。

## なぜ規約が必要か

1. **チーム開発の効率化** - 誰が書いても同じスタイルで統一される
2. **検索性の向上** - 命名パターンが決まっているため、すぐに見つかる
3. **バグの削減** - 一貫性があることでタイプミスや混乱を防ぐ
4. **オンボーディングの高速化** - 新メンバーがルールを学びやすい

## ファイル命名規約

すべてのファイル名は **kebab-case** (小文字 + ハイフン) を使用します。

### ルール

| 対象 | 規約 | 例 |
|------|------|-----|
| **MDXファイル** | kebab-case.mdx | `search-autocomplete.mdx`, `data-table.mdx` |
| **TypeScript/JavaScript** | kebab-case.ts(x) | `use-local-storage.ts`, `api-client.tsx` |
| **CSSファイル** | kebab-case.css | `global-styles.css`, `button-variants.css` |
| **設定ファイル** | kebab-case.json | `tsconfig.json`, `package.json` |

### 良い例

```
✓ search-autocomplete.mdx
✓ data-table.tsx
✓ use-debounce.ts
✓ global-styles.css
✓ api-client.js
```

### 悪い例

```
✗ SearchAutocomplete.mdx  (PascalCase)
✗ data_table.tsx          (snake_case)
✗ useDebounce.ts          (camelCase)
✗ GlobalStyles.css        (PascalCase)
✗ APIClient.js            (PascalCase)
```

## コンポーネント命名規約

コンポーネント名は **PascalCase** を使用します。

### ルール

```typescript
// ✓ 良い例: PascalCase
export function SearchBar() { }
export const DataTable = () => { }
export class UserProfile extends Component { }

// ✗ 悪い例: camelCase, snake_case
export function searchBar() { }
export const data_table = () => { }
export class user_profile extends Component { }
```

### コンポーネントとファイル名の対応

| ファイル名 | コンポーネント名 |
|-----------|----------------|
| `search-bar.tsx` | `SearchBar` |
| `data-table.tsx` | `DataTable` |
| `user-profile.tsx` | `UserProfile` |

## CSS クラス命名規約

CSS クラス名は **BEM** (Block Element Modifier) またはユーティリティベースを推奨します。

### BEM 記法

```
Block__Element--Modifier
```

#### 例

```css
/* Block */
.card { }

/* Element */
.card__header { }
.card__body { }
.card__footer { }

/* Modifier */
.card--highlighted { }
.card__header--large { }
```

### ✓ 良い例

```html
<div class="card card--highlighted">
  <div class="card__header card__header--large">
    タイトル
  </div>
  <div class="card__body">
    コンテンツ
  </div>
</div>
```

### ユーティリティクラス

```css
/* ✓ 良い例: kebab-case */
.text-center { text-align: center; }
.flex-col { flex-direction: column; }
.bg-primary { background: #3b82f6; }

/* ✗ 悪い例: camelCase */
.textCenter { text-align: center; }
.flexCol { flex-direction: column; }
```

## JavaScript/TypeScript 変数・関数命名規約

### 変数・関数: camelCase

```typescript
// ✓ 良い例
const userName = "John";
let currentPage = 1;
function getUserData() { }
const handleClick = () => { };

// ✗ 悪い例
const UserName = "John";      // PascalCase
let current_page = 1;         // snake_case
function GetUserData() { }    // PascalCase
```

### クラス: PascalCase

```typescript
// ✓ 良い例
class ApiClient { }
class UserManager { }

// ✗ 悪い例
class apiClient { }
class user_manager { }
```

### 定数: UPPER_SNAKE_CASE

```typescript
// ✓ 良い例
const API_BASE_URL = "https://api.example.com";
const MAX_RETRIES = 3;
const DEFAULT_TIMEOUT = 5000;

// ✗ 悪い例
const apiBaseUrl = "https://api.example.com";
const maxRetries = 3;
const defaultTimeout = 5000;
```

### 型・インターフェース: PascalCase

```typescript
// ✓ 良い例
interface UserData { }
type ApiResponse = { };
enum Status { }

// ✗ 悪い例
interface userData { }
type apiResponse = { };
enum status { }
```

## タグ規約

すべてのコンテンツには適切なタグを付与します。

### タグ付けルール

| ルール | 説明 |
|--------|------|
| **必須** | すべてのMDXファイルに tags フィールドが必須 |
| **最大5個** | 1つのファイルに最大5個まで |
| **小文字のみ** | すべて小文字 (ui, form, animation) |
| **ハイフン可** | 複合語はハイフン区切り (css-grid, web-animations) |
| **アルファベット順** | 配列内は必ずアルファベット順にソート |
| **重複禁止** | 同じタグを重複して記載しない |

### 統制語彙 (推奨タグリスト)

#### カテゴリタグ

```yaml
- ui              # UIコンポーネント
- form            # フォーム関連
- animation       # アニメーション
- performance     # パフォーマンス最適化
- accessibility   # アクセシビリティ
- responsive      # レスポンシブデザイン

```

### カスタムバリデーションスクリプト

```javascript
// scripts/validate-naming.js
const fs = require('fs');
const path = require('path');

function validateFileName(filePath) {
  const fileName = path.basename(filePath, path.extname(filePath));
  const kebabCaseRegex = /^[a-z0-9]+(-[a-z0-9]+)*$/;

  if (!kebabCaseRegex.test(fileName)) {
    console.error(\`❌ Invalid file name: \${filePath}\`);
    console.error(\`   Expected kebab-case, got: \${fileName}\`);
    return false;
  }

  return true;
}

function validateFrontmatter(filePath) {
  const content = fs.readFileSync(filePath, 'utf-8');
  const frontmatterRegex = /^---\n([\s\S]+?)\n---/;
  const match = content.match(frontmatterRegex);

  if (!match) {
    console.error(\`❌ No frontmatter found: \${filePath}\`);
    return false;
  }

  const frontmatter = match[1];
  const tagsMatch = frontmatter.match(/tags:\s*\[(.+?)\]/);

  if (!tagsMatch) {
    console.error(\`❌ Missing tags: \${filePath}\`);
    return false;
  }

  const tags = tagsMatch[1].split(',').map(t => t.trim());

  // Check max 5 tags
  if (tags.length > 5) {
    console.error(\`❌ Too many tags (\${tags.length}): \${filePath}\`);
    return false;
  }

  // Check alphabetical order
  const sortedTags = [...tags].sort();
  if (JSON.stringify(tags) !== JSON.stringify(sortedTags)) {
    console.error(\`❌ Tags not sorted: \${filePath}\`);
    console.error(\`   Current: \${tags.join(', ')}\`);
    console.error(\`   Expected: \${sortedTags.join(', ')}\`);
    return false;
  }

  return true;
}

// Run validation
const contentDir = 'src/content';
let hasErrors = false;

fs.readdirSync(contentDir, { recursive: true }).forEach(file => {
  if (file.endsWith('.mdx')) {
    const filePath = path.join(contentDir, file);
    if (!validateFileName(filePath) || !validateFrontmatter(filePath)) {
      hasErrors = true;
    }
  }
});

if (hasErrors) {
  process.exit(1);
} else {
  console.log('✅ All naming conventions are valid');
}
```

## よくある間違いと修正方法

### 間違い1: ファイル名とコンポーネント名の不一致

```typescript
// ❌ 悪い例
// ファイル: SearchBar.tsx
export function searchBar() { }

// ✅ 良い例
// ファイル: search-bar.tsx
export function SearchBar() { }
```

### 間違い2: タグが多すぎる

```yaml
# ❌ 悪い例
tags: [ui, react, typescript, form, input, validation, accessibility]  # 7個

# ✅ 良い例
tags: [accessibility, form, react, typescript, ui]  # 5個
```

### 間違い3: タグがソートされていない

```yaml
# ❌ 悪い例
tags: [react, ui, typescript, form]

# ✅ 良い例
tags: [form, react, typescript, ui]  # アルファベット順
```

### 間違い4: CSS クラスの命名ミス

```css
/* ❌ 悪い例 */
.cardHeader { }
.card_header { }
.Card-Header { }

/* ✅ 良い例 */
.card__header { }      /* BEM */
.card-header { }       /* ユーティリティ */
```

## まとめ

命名規約とタグ規約を守ることで：

1. **検索性向上** - ファイルやコンポーネントがすぐ見つかる
2. **一貫性確保** - プロジェクト全体で統一された命名
3. **メンテナンス容易** - 将来の変更が簡単になる
4. **チーム協調** - 誰が書いても同じスタイル

必ず上記のチェックリストとバリデーションツールを使って、規約を守りましょう。
