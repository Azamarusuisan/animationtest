## 60fpsを維持するための必須知識

Webアニメーションをスムーズに動かすには、1フレームを**16.67ms以内**（60fps）で処理する必要があります。ブラウザのレンダリングパイプラインを理解し、適切なプロパティを使うことが重要です。

### ブラウザのレンダリングパイプライン

```
JavaScript → Style → Layout → Paint → Composite
```

- **JavaScript**: アニメーション処理の実行
- **Style**: CSS スタイルの計算
- **Layout**: 要素の配置とサイズの計算（最も重い処理）
- **Paint**: ピクセルの描画
- **Composite**: レイヤーの合成（GPU で処理）

## パフォーマンスに影響するプロパティ

### 最速: Composite のみ（GPU アクセラレーション）

これらのプロパティのみを変更すれば、GPU で処理されるため最も高速です：

```css
/* GPU で処理される高速なプロパティ */
.element {
  transform: translate(10px, 20px);
  transform: scale(1.5);
  transform: rotate(45deg);
  opacity: 0.5;
}
```

### 遅い: Layout + Paint + Composite

これらのプロパティを変更すると、レイアウトの再計算が発生し、パフォーマンスが低下します：

```css
/* 避けるべき重いプロパティ */
.element {
  width: 200px;      /* Layout 発生 */
  height: 100px;     /* Layout 発生 */
  top: 50px;         /* Layout 発生 */
  left: 100px;       /* Layout 発生 */
  margin: 20px;      /* Layout 発生 */
  padding: 10px;     /* Layout 発生 */
  font-size: 16px;   /* Layout 発生 */
}
```

### 1. translate3d() を使う

```css
/* ❌ 2D 変換: GPU が使われない場合がある */
.element {
  transform: translateY(10px);
}

/* ✅ 3D 変換: 確実に GPU レイヤーを作成 */
.element {
  transform: translate3d(0, 10px, 0);
}
```

```css
/* ❌ 悪い例: 常時適用（メモリを消費） */
.element {
  will-change: transform, opacity;
}

/* ✅ 良い例: ホバー時のみ */
.element:hover {
  will-change: transform;
}

.element {
  transition: transform 0.3s;
}

/* ✅ JavaScript で動的に制御 */
element.addEventListener('mouseenter', () => {
  element.style.willChange = 'transform';
});

element.addEventListener('animationend', () => {
  element.style.willChange = 'auto'; // 完了後に解除
});
```

### 3. backface-visibility

3D変換を使わない場合でも、GPU レイヤーを作成する方法：

```css
.element {
  backface-visibility: hidden;
  /* または */
  -webkit-backface-visibility: hidden;
}
```

## パフォーマンス計測

### Chrome DevTools での確認

1. **Performance タブ**
   - Record ボタンをクリック
   - アニメーションを実行
   - Stop をクリック
   - FPS グラフを確認（60fps を維持しているか）

2. **Rendering タブ**
   - "Paint flashing": 再描画をハイライト表示
   - "Layout Shift Regions": レイアウトシフトを可視化
   - "Frame Rendering Stats": FPS を常時表示

3. **Layers タブ**
   - どの要素がレイヤー化されているか確認
   - 不要なレイヤーがないかチェック

### JavaScript で FPS を計測

```javascript
let fps = 0;
let lastTime = performance.now();
let frameCount = 0;

function measureFPS() {
  const now = performance.now();
  frameCount++;

  // 1秒ごとに FPS を計算
  if (now >= lastTime + 1000) {
    fps = Math.round((frameCount * 1000) / (now - lastTime));
    frameCount = 0;
    lastTime = now;
    console.log('FPS:', fps);

    if (fps < 55) {
      console.warn('⚠️ パフォーマンス低下検出');
    }
  }

  requestAnimationFrame(measureFPS);
}

measureFPS();
```

## ベストプラクティス

### チェックリスト

- [ ] `transform` と `opacity` のみでアニメーション
- [ ] `translate3d()` で GPU アクセラレーション
- [ ] `requestAnimationFrame` を使用
- [ ] `will-change` は必要な時だけ適用
- [ ] アニメーション完了後に `will-change: auto` で解除
- [ ] `prefers-reduced-motion` に対応
- [ ] DevTools で 60fps を確認
- [ ] モバイルデバイスでテスト

### 大量の要素をアニメーション

```javascript
// ❌ 悪い例: 個別にアニメーション
items.forEach(item => {
  item.style.transform = `translateY(${y}px)`;
});

// ✅ 良い例: 親要素をアニメーション
container.style.transform = `translateY(${y}px)`;
```

### アクセシビリティへの配慮

```css
/* アニメーションを無効化する設定のユーザーに配慮 */
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

## まとめ

1. **transform と opacity のみを使用** - Layout を回避し、Composite のみで処理
2. **translate3d() で GPU アクセラレーション** - 確実にハードウェアアクセラレーションを有効化
3. **will-change は必要な時だけ** - メモリ消費を抑えるため、適切に管理
4. **60fps を維持** - DevTools で定期的にパフォーマンスを確認
5. **アクセシビリティに配慮** - prefers-reduced-motion に必ず対応

これらを守れば、ほとんどのアニメーションで 60fps を維持できます。
